  • Given that a user invokes "Create Bucket"

Create a Bucket in the bucket page using valid token

  • When the user invokes the Create Bucket API using the Goal Title as <Goal_Title>, Goal Description as <Goal_Description> and Goal Date as <Goal_Date> using token <file:/resources/access_tokens/valid_token.txt>
  • Then the status code for creating a bucket request is "201"
  • And the JSON Path Assertions for the response of creating a bucket request contains the following
    JSON Path Expected Result
    $.title <Goal_Title>
    $.description <Goal_Description>
    $.goalDate <Goal_Date>

Create a Bucket in the bucket page using invalid token

  • When the user invokes the Create Bucket API using the Goal Title as <Goal_Title>, Goal Description as <Goal_Description> and Goal Date as <Goal_Date> using token <file:/resources/access_tokens/invalid_token.txt>
  • Then the status code for creating a bucket request is "401"

Create a Bucket in the bucket page using no token

  • When the user invokes the Create Bucket API using the Goal Title as <Goal_Title>, Goal Description as <Goal_Description> and Goal Date as <Goal_Date> using token <file:/resources/access_tokens/null_token.txt>
  • Then the status code for creating a bucket request is "401"

Create a Bucket in the bucket page using past date

  • When the user invokes the Create Bucket API using the Goal Title as <Goal_Title>, Goal Description as <Goal_Description> and Goal Date as "2017-06-21T00:00:00" using token <file:/resources/access_tokens/valid_token.txt>
  • Then the status code for creating a bucket request is "400"
  • And the JSON Path Assertions for the response of creating a bucket request contains the following
    JSON Path Expected Result
    $.GoalDate[0] Goal date should be a future date
